Multi-Tenancy Architecture

📊 Database-Per-Tenant Strategy

Catalog Database: Central registry storing all tenant metadata, connection strings, and user-tenant mappings
Tenant Databases: Each organization gets an isolated database (ezofis_Tenant_{number})
Tenant Resolution: X-Tenant-Id header or JWT 'tid' claim → Middleware resolves connection string

Request Processing Pipeline

Incoming Request ↓ 1. HTTPS Redirection → Enforce secure connections ↓ 2. Secure Headers → Add security headers (HSTS, CSP, X-Frame-Options) ↓ 3. Correlation ID → Generate unique request ID for tracing ↓ 4. CORS → Handle cross-origin requests ↓ 5. Authentication → Validate JWT token ↓ 6. Authorization → Check user roles and policies ↓ 7. TenantConnection Middleware → Resolve tenant database ↓ 8. Controller → Execute business logic (CQRS) ↓ Response with X-Correlation-ID
